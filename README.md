# Bike Rental System

Bike Rental System is a Spring Boot web application for managing bike rentals, ride bookings, ride sharing, and user accounts. The project supports three user roles:

- `ADMIN` manages platform users
- `OPERATOR` manages bikes and sees bookings for bikes they own
- `RIDER` browses bikes, creates bookings, and can request to share another rider's trip

The application uses:

- Java 8
- Spring Boot 2.7
- Thymeleaf templates for the UI
- Bootstrap and custom CSS for styling
- Text files for persistence instead of a database

## Main Features

- User registration and login
- Role-based dashboard routing
- Bike listing on the public landing page
- Operator bike CRUD operations
- Rider booking creation, update, and deletion
- Ride-sharing request flow between riders
- Admin user search, create, update, and delete
- File-based data storage for users, bikes, and bookings

## Folder Explanation

### `src/main/java/app`

Contains the Spring Boot entry point and application-level packages.

- `BikeRentalSystemApplication.java`
  Starts the application and allows WAR deployment through `SpringBootServletInitializer`.

### `src/main/java/app/controller`

Contains the MVC controllers that handle HTTP requests and decide which page to render or where to redirect the user.

- `BaseWebController.java`
  Shared helper logic for controllers. It reads the current user from the session, resolves the data directory, and handles role-based redirects.

- `PageController.java`
  Handles page navigation such as home, login, register, add-bike, and edit pages.

- `AuthController.java`
  Handles login, registration, and logout.

- `BikeController.java`
  Handles operator bike management such as viewing bikes, adding bikes, updating bikes, and deleting bikes.

- `RideBookingController.java`
  Handles rider bookings, ride updates, ride deletion, and ride-sharing requests.

- `UserController.java`
  Handles admin-only user management.

### `src/main/java/app/support`

Contains helper classes used by controllers or business logic.

- `BikeStatusHelper.java`
  Synchronizes bike status with the state of ride bookings.

### `src/main/java/dao`

Contains the data access layer. These classes read and write plain text files.

- `UserDAO.java`
  Stores and retrieves users from `users.txt`

- `BikeDAO.java`
  Stores and retrieves bikes from `bikes.txt`

- `RideBookingDAO.java`
  Stores and retrieves bookings from `ride-bookings.txt`

- `DataDirectoryResolver.java`
  Resolves the active data folder. It prefers `src/main/webapp/data` when the project is run from the source tree.

### `src/main/java/model`

Contains the domain model used by the application.

- `Person.java`
  Base class for credentials and shared user data

- `User.java`
  Abstract base class for all user types

- `AdminUser.java`, `OperatorUser.java`, `RiderUser.java`
  Role-specific user implementations

- `UserFactory.java`
  Creates the correct user type from role names

- `Bike.java`
  Represents a bike in the system

- `RideBooking.java`
  Represents a bike booking and its ride details

- `ShareRideRequest.java`
  Represents a rider's request to join an existing ride

### `src/main/resources/templates`

Contains the frontend HTML views used by Spring Boot and Thymeleaf.

- `index.html`
  Landing page and public bike list

- `login.html`
  Login page

- `register.html`
  Registration page

- `bikes.html`
  Operator bike management page

- `add-bike.html`
  Form to add a bike

- `edit-bike.html`
  Form to edit a bike

- `ride-bookings.html`
  Rider or operator booking page, depending on the logged-in role

- `edit-ride-booking.html`
  Form to update an existing booking

- `users.html`
  Admin user management page

- `edit-user.html`
  Form to update a user account

### `src/main/resources/static/assets`

Contains static frontend assets served directly by Spring Boot.

- `app-theme.css`
  Main application styling

- `img.png`
  Logo or image asset used in the UI

### `src/main/webapp/data`

Contains the text files used as the application's persistent storage.

- `users.txt`
  User records

- `bikes.txt`
  Bike records

- `ride-bookings.txt`
  Booking records including ride-sharing information

### `target`

Contains generated files after building the project. This is not source code and should not be edited manually.

## How the Project Works

### 1. Application Startup

The application starts from `BikeRentalSystemApplication.java`. Spring Boot scans the `app` package, registers controllers, and serves the HTML templates from `src/main/resources/templates`.

### 2. Authentication and Session Handling

- Users log in through `AuthController`
- On successful login, the authenticated user is saved in the HTTP session as `currentUser`
- After login, the user is redirected based on role:
  - `ADMIN` -> `/users`
  - `OPERATOR` -> `/bikes`
  - `RIDER` -> `/rideBookings`

### 3. Public Landing Page

The landing page is handled by `PageController.home()`.

It:

- loads all bikes from `BikeDAO`
- filters available bikes
- shows public fleet information
- displays a role-aware dashboard link if a user is logged in

### 4. Bike Management

Operators use `BikeController` to:

- view bikes
- add a bike
- update bike details
- delete a bike

Each bike stores:

- id
- bike name
- type
- station
- hourly rate
- status
- operator username

Operators can only edit or delete their own bikes.

### 5. Ride Booking Flow

Riders use `RideBookingController` to:

- browse bookable bikes
- create bookings
- edit their own bookings
- delete their own bookings

When a rider books a bike:

- the booking is saved into `ride-bookings.txt`
- the selected bike status is updated to `BOOKED`

When a booking is edited or deleted, `BikeStatusHelper` updates the bike status to stay consistent with the booking state.

### 6. Ride Sharing

Riders can request to join other riders' ongoing rides.

Flow:

1. A rider opens the ride bookings page
2. The system shows ongoing rides that do not belong to that rider
3. The rider submits a share request with pickup and stop points
4. The owner of the original ride can accept or decline the request
5. Accepted requests are stored inside the related booking record

This feature is implemented with:

- `RideBooking`
- `ShareRideRequest`
- `RideBookingController`
- `RideBookingDAO`

### 7. User Management

Admins use `UserController` to:

- list users
- search users
- create users
- update users
- delete users

The admin is prevented from deleting their own currently logged-in account through the normal delete action.

## Persistence Design

This project does not use MySQL or another relational database. Instead, it stores data in text files.

Advantages:

- simple to understand
- easy to move between computers
- no separate database setup required

Tradeoffs:

- limited scalability
- no transaction support
- manual parsing of file content
- not suitable for high-concurrency production systems

### Data Files

- `users.txt` uses `username:password:role`
- `bikes.txt` stores bike fields separated by `:`
- `ride-bookings.txt` stores booking fields plus serialized share-request data

The DAO layer parses each line and reconstructs Java objects when the application reads data.

## Frontend Structure

The frontend is built with server-rendered HTML templates, not JSP.

- HTML views are in `src/main/resources/templates`
- CSS and images are in `src/main/resources/static/assets`
- Bootstrap is loaded from CDN
- Thymeleaf is used to bind dynamic values into the templates

This means the browser receives rendered HTML after the server processes the template.

## Important Notes

- The real frontend source files are the `.html` templates under `src/main/resources/templates`
- `target/` is generated output only
- `src/main/webapp/data` is used for persistence even though the UI templates are under `resources/templates`
- Because the system uses text files, editing or deleting records directly in the data files affects the application immediately

## Summary

This project is a role-based bike rental and ride-sharing platform built with Spring Boot. Its backend is organized into controllers, DAOs, models, and helpers. Its frontend is built with Thymeleaf-rendered HTML templates and static assets. Instead of a database, it persists users, bikes, and ride bookings in text files stored inside the project.
