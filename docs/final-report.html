1. Introduction

This project is a web-based Bike Rental and Ride-Sharing Platform developed for the SE1020 Object Oriented Programming module. The system allows users to register, log in, manage bikes, create ride bookings, and handle ride-sharing requests. The backend is implemented using Java and Spring Boot, while file handling is used for data persistence instead of a database.

  • Implement CRUD operations for user management, bike management, and ride bookings.

6. Object-Oriented Programming Concepts

Encapsulation

Encapsulation is used in the model classes through controlled access to data fields using getters and setters. Examples include Bike, RideBooking, and ShareRideRequest.

Inheritance

Inheritance is used through the class hierarchy Person -> User -> AdminUser / OperatorUser / RiderUser. This structure promotes reuse of common properties such as username and password.

Polymorphism

Polymorphism is implemented using specialized user subclasses. The system works with the abstract User type, while each subclass provides its own implementation of getRole() and getDashboardPath(). This allows the application to redirect users based on behavior defined by the actual runtime object.

Abstraction

The classes Person and User are abstract and define the common structure and required methods for all user types.

8. File Handling

The application stores data using text files instead of a relational database.

  • users.txt - stores usernames, passwords, and roles
  • bikes.txt - stores bike records
  • ride-bookings.txt - stores ride booking and share request details

File operations are handled through UserDAO, BikeDAO, and RideBookingDAO.
